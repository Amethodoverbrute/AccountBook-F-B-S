# 项目优化计划

## 优化范围

根据README.md中列出的不足与优化方向，选择以下6项进行优化：
1. 功能不完善：添加账单编辑功能、数据统计和图表展示功能、账单分类管理功能
4. 文档不完善：添加Swagger API文档
6. 代码冗余：清理未使用的Book模型
8. 缺少日志记录：添加日志记录功能
9. UI设计待优化：优化页面设计，提升用户体验
10. 性能优化：优化数据库查询和前端渲染性能

## 详细优化计划

### 1. 清理未使用的Book模型（后端）

**实现步骤**：
- 删除`backend/models/bookModel.js`文件
- 删除相关路由和控制器（如果存在）
- 清理其他文件中对Book模型的引用

**预期效果**：减少代码冗余，提高项目维护性

### 2. 添加日志记录功能（后端）

**技术选型**：winston（强大的日志库）

**实现步骤**：
- 安装winston依赖
- 创建日志配置文件
- 在中间件中添加请求日志记录
- 在关键代码处添加业务日志
- 配置日志级别和输出格式

**预期效果**：便于调试和监控，提高系统可维护性

### 3. 完善API文档（后端）

**技术选型**：swagger-jsdoc + swagger-ui-express

**实现步骤**：
- 安装swagger相关依赖
- 创建Swagger配置文件
- 在API路由中添加Swagger注释
- 集成Swagger UI到Express应用
- 测试和验证API文档

**预期效果**：自动生成交互式API文档，便于前后端协作

### 4. 添加账单编辑功能（前后端）

**实现步骤**：
- **前端**：
  - 在Home.vue中添加编辑按钮
  - 创建编辑表单组件或复用添加表单
  - 实现编辑状态管理
  - 添加编辑相关的样式
- **后端**：
  - 验证现有的PATCH /api/account/:id接口
  - 确保接口正确处理编辑请求

**预期效果**：用户可以编辑已添加的账单，提高功能完整性

### 5. 添加数据统计和图表展示功能（前后端）

**技术选型**：ECharts（强大的图表库）

**实现步骤**：
- **后端**：
  - 添加统计API，支持按时间、类型等维度统计
  - 实现数据聚合逻辑
- **前端**：
  - 安装ECharts依赖
  - 创建统计页面或组件
  - 实现图表渲染和交互
  - 添加统计筛选功能

**预期效果**：用户可以查看账单统计数据和图表，提升数据可视化能力

### 6. 添加账单分类管理功能（前后端）

**实现步骤**：
- **后端**：
  - 创建分类模型
  - 添加分类CRUD API
  - 更新账单模型，关联分类
- **前端**：
  - 添加分类管理页面
  - 在账单添加/编辑表单中添加分类选择
  - 实现分类的增删改查功能

**预期效果**：用户可以自定义账单分类，提高账单管理的灵活性

### 7. UI设计优化（前端）

**实现步骤**：
- 优化页面布局和配色方案
- 改进账单卡片样式
- 添加动画效果，提升交互体验
- 优化表单设计，提高用户体验
- 完善响应式设计，适配更多设备

**预期效果**：页面更加美观，用户体验更好

### 8. 性能优化

**实现步骤**：
- **后端**：
  - 为数据库添加索引（如userId、time等字段）
  - 优化查询语句，避免全表扫描
  - 实现数据分页
- **前端**：
  - 优化组件渲染，避免不必要的重渲染
  - 实现数据缓存
  - 优化图片和资源加载

**预期效果**：系统响应更快，性能更好
